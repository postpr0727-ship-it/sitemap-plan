# Firebase 설정 가이드 (간단 버전)

**목표**: 크롬, 사파리 등 어떤 브라우저에서든 같은 프로젝트 데이터를 볼 수 있게 하기

> 💡 **참고**: 이 가이드는 **클라이언트 사이드 방식**입니다.  
> 서버 사이드 방식과의 차이는 `FIREBASE_COMPARISON.md` 파일을 참고하세요.

## 3단계로 끝내기

### 1️⃣ Firebase 프로젝트 만들기
1. [Firebase Console](https://console.firebase.google.com) 접속
2. "프로젝트 추가" 클릭 → 이름 입력 → 만들기
3. 왼쪽 메뉴: **빌드** → **Realtime Database** → **데이터베이스 만들기**
4. 위치 선택 (서울 권장) → **테스트 모드에서 시작** → **사용 설정**

### 1-1. ⚠️ 보안 규칙 설정 (중요!)
1. Firebase Console에서 **Realtime Database** 메뉴 클릭
2. 상단 탭에서 **"규칙"** 클릭
3. 다음 규칙으로 교체:
```json
{
  "rules": {
    "sitemapProjects": {
      ".read": true,
      ".write": true
    }
  }
}
```
4. **"게시"** 버튼 클릭

### 2️⃣ 설정 정보 가져오기
1. 왼쪽 상단 **톱니바퀴 아이콘** → **프로젝트 설정**
2. 아래로 스크롤 → **웹 아이콘 `</>`** 클릭
3. 앱 이름 입력 → **앱 등록**
4. 나타나는 설정 정보 **전체 복사**

### 3️⃣ 코드에 붙여넣기
1. `index.html` 파일 열기
2. `firebaseConfig` 찾기 (약 607줄)
3. 복사한 설정 정보로 **전체 교체**
4. 저장 후 브라우저 새로고침

## 확인하기

브라우저에서 `index.html` 열고:
- 개발자 도구(F12) → Console에 "Firebase가 성공적으로 초기화되었습니다." 표시
- 헤더에 **🟢 실시간 동기화** 표시

## 완료! 이제 가능한 것들

## 문제 해결

**"Firebase 설정이 필요합니다"가 계속 나와요**
→ `firebaseConfig`의 값들이 여전히 `YOUR_API_KEY` 같은지 확인하세요.

**실시간 동기화가 안 돼요**
→ Firebase Console에서 Realtime Database가 활성화되어 있는지 확인하세요.

**"PERMISSION_DENIED: Permission denied" 오류가 나와요** ⚠️
→ **가장 흔한 오류입니다!** Firebase Console에서:
1. **Realtime Database** → **규칙** 탭으로 이동
2. 위의 "1-1. 보안 규칙 설정" 단계를 따라하세요
3. `sitemapProjects` 경로에 읽기/쓰기 권한을 허용해야 합니다
4. 규칙을 수정한 후 **"게시"** 버튼 클릭
